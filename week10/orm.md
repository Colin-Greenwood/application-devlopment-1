What problems does raw SQL create in large applications?

Raw SQL can become hard to manage as applications grow because queries are repeated, harder to read, and tightly tied to the database structure. It also increases the risk of bugs and security issues like SQL injection if not handled properly.

What is an ORM in your own words?

An ORM (Object Relational Mapper) is a tool that lets developers interact with a database using code instead of writing raw SQL queries.

What does an ORM replace or simplify?

It replaces manual SQL queries with methods and objects, simplifies database interactions, and helps organize data logic in a cleaner and more reusable way.

When would you NOT want to use an ORM?

You might avoid using an ORM when performance is critical, when you need very complex or optimized queries, or when you want full control over the SQL being executed.
